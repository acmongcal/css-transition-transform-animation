CSS Transitions

What is CSS Transitions?

CSS Transition is a property that allows developers to change property values smoothly over a specified duration, creating a gradual effect instead of an instant change.

Basic Transition Syntax

Hover Me

          .element {
                background-color: #ff7a59;
                transition: background-color 0.5s ease;
            }
          .element:hover {
                background-color: #33475b;
            }

What Does It Do?

CSS Transition allows you to change property values smoothly, from an initial state to an end state, typically triggered by user interactions like hover or focus.

The 4 Core Transition Properties

To control exactly how the transition behaves, it utilizes these four sub-properties:

  • transition-property: Specifies which styles to change (e.g., background-color, width, opacity). 
    transition-property: background-color;
  • transition-duration: Sets how long the effect takes to complete (e.g., 0.5s). 
    transition-duration: 0.5s;
  • transition-timing-function: Controls the speed curve of the transition (e.g., ease-in-out, linear). 
    transition-timing-function: ease-in-out;
  • transition-delay: Defines the waiting time before the transition actually starts. 
    transition-delay: 0.1s;

Common Triggers

Transitions require a trigger to start. They cannot play automatically on page load:

  • :hover (Mouse over)
  • :focus (Keyboard navigation or input selection)
  • :active (Clicking down on an element)

Why Would You Use It?

CSS transitions are the easiest and most performant way to create subtle interactive responses without JS.

  • Smooth Hover Effects: Prevents sudden, jarring color and size changes when users interact.
  • Better Visual Feedback: Clearly signals to users that an element is clickable or active.
  • No Layout Shift: Animates elements smoothly without breaking or moving the surrounding layout.

Pros and Cons

Pros

  • Performance: Hardware-accelerated by the GPU, ensuring smooth rendering without layout jank.
  • Simplicity: Easy to implement and perfect for simple two-state micro-interactions.
  • Maintainability: Written directly in CSS, keeping styles separate from JavaScript logic.

Cons

  • Limited States: Can only transition from State A to State B, making complex animations impossible.
  • Trigger Required: Cannot run automatically on page load or loop infinitely without JavaScript.
  • Property Restrictions: Certain CSS properties (like display: none to block) cannot be smoothly transitioned natively.

1. Longhand Approach

          
            .transition-box {
              background-color: #ff7a59;

              transition-property: background-color;
              transition-duration: 0.5s;
              transition-timing-function: ease-in-out;
              transition-delay: 0.1s;
            }

2. Shorthand Approach

          
            .transition-box {
              background-color: #ff7a59;

              /* Order: property | duration | timing-function | delay */
              transition: background-color 0.5s ease-in-out 0.1s;
            }

            .transition-box:hover {
              background-color: #33475b;
            }
